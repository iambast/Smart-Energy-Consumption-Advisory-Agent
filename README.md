## Project Overview

This project develops an **AI-powered agent** that analyses household smart meter data to detect usage patterns, identify anomalies, and deliver personalised energy-saving recommendations. The system uses unsupervised machine learning (K-Means, Isolation Forest, DBSCAN) and a rule-based recommendation engine to advise households on reducing peak-hour consumption and lowering electricity costs.

**Dataset:** Smart meter dataset (UCI / Kaggle) — 15-minute interval readings including active power, sub-metering channels, voltage, and current intensity.

## Notebook Descriptions

### Notebook 1 — Data Understanding and Preprocessing
Generates a synthetic smart meter dataset (mirroring UCI schema), audits missing values, applies forward/backward fill imputation, removes outliers via IQR fencing, engineers temporal features (hour, day, cyclical encodings), and normalises all numeric columns using MinMaxScaler.

### Notebook 2 — Feature Engineering
Builds rolling window statistics (1-hour and 24-hour means/std), lag features (15-min, 1-hour, 24-hour), time-of-use (ToU) segments with one-hot encoding, daily aggregate profiles, and a PCA variance analysis to understand feature importance.

### Notebook 3 — Unsupervised Modelling
Applies K-Means clustering (with elbow method and silhouette score to select k=4), Isolation Forest for anomaly detection (~2% contamination), and DBSCAN on PCA-projected data for density-based pattern discovery.

### Notebook 4 — Visualization and Analysis
Produces a day×hour heatmap of average consumption, cluster diurnal profiles, monthly energy trend bars, sub-metering pie chart, daily anomaly calendar, and a stacked cluster-by-day-of-week bar chart.

### Notebook 5 — Interpretation, Recommendation Engine & Ethics
Maps clusters to human-readable behavioural labels, implements a rule-based recommendation engine with personalised tips for each risk level, performs a batch risk assessment over the full year, and critically analyses privacy, fairness, transparency, and societal impact.

## Notes

- The dataset is **synthetically generated** within Notebook 1 to mirror the UCI smart meter schema. To use real UCI/Kaggle data, replace the data generation cell in Notebook 1 with a `pd.read_csv()` call pointing to your downloaded file, ensuring the column names match those expected.
- All notebooks are **fully commented** — every line of code includes an explanation of its purpose and contribution to the analysis.
- The presentation file (`Smart_Energy_Advisory_Agent.pptx`) provides a 6-slide visual summary for demonstration purposes.
